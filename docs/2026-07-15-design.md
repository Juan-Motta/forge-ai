# Design record

- **Date:** 2026-07-15
- **Goal:** interoperable workflow discipline for Claude Code, Codex, and OpenCode, in the
  simplest possible form — skills + config only, no hooks/scripts.

## Decisions

| Decision | Choice | Why |
| --- | --- | --- |
| Harness support | Claude Code, Codex, **and** OpenCode, interchangeable | Interoperability is a hard requirement |
| Enforcement | **Advisory** + native coarse approval on push/PR | Simplest; no scripts. Trades away conditional hard-blocking |
| Canonical instruction | `CLAUDE.md`, with `AGENTS.md` → symlink | One source; all three harnesses read it |
| Canonical skills | `skills/<name>/SKILL.md`, symlinked to `.claude/skills`, `.codex/skills`, `.opencode/skills` | `SKILL.md` is a convention all three share |
| Discipline | `shared/rules/*.md` | Harness-agnostic markdown |
| State | `.workflow/state.md` (from `state.template.md`) | Source of truth for the ship-gate checklist |

## Verified facts (this machine + official docs)

- Codex reads discipline from `AGENTS.md` (root); repo skills under `<repo>/.codex/skills`
  (global `~/.codex/skills`); `approval_policy` is the native, script-free approval gate.
- Claude Code `ask`-tier permissions natively prompt on `git push` / `gh pr create`.
- OpenCode auto-reads `AGENTS.md` (fallback `CLAUDE.md`) with no config; discovers skills
  at `.opencode/skills`, `.claude/skills`, and `.agents/skills` (so it reads the shared
  folder directly); gates commands via `opencode.json` `permission.bash` (per-pattern
  `ask`/`deny`, last-match-wins, patterns need a trailing `*`).

## Deliberate trade (know this)

No hook conditionally blocks a commit when gates are unmet. Enforcement = advisory
(instructions) + native approval prompt on outward actions (human backstop). This starts
as simple as possible.

## Per-engine config (the only non-shared parts)

- Claude Code: `.claude/settings.json` (`ask`-tier push/PR, deny force-push).
- Codex: `.codex/config.toml` (`approval_policy`).
- OpenCode: `opencode.json` (`permission.bash`: push/PR `ask`, force-push `deny`).

The permission syntax is **not portable** across the three — each engine keeps its own
config file. Everything else (instructions, skills, rules) is one shared source.

## Explicitly deferred (future phase, if hard blocking is wanted)

Hook-based conditional blocking (deny commit/push/PR when a `.workflow/state.md` box is
unchecked). Feasible on all three, but it reintroduces adapter/trust/merge complexity that
is deliberately out of scope for this build.

## Open items to verify with a live run

- Confirm each harness discovers the **symlinked** skills dir and symlinked root `AGENTS.md`.
- Confirm `SKILL.md` frontmatter (`name`, `description`) is accepted by all three.
- If a root `AGENTS.override.md` ever exists, it takes precedence over `AGENTS.md` — the
  discipline symlink must target the active file.
