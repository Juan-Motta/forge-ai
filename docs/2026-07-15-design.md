# Design record

- **Date:** 2026-07-15
- **Goal:** interoperable workflow discipline for Claude Code + Codex, in the simplest
  possible form — skills + config only, no hooks/scripts.

## Decisions

| Decision | Choice | Why |
| --- | --- | --- |
| Harness support | Claude Code **and** Codex, interchangeable | Interoperability is a hard requirement |
| Enforcement | **Advisory** + native coarse approval on push/PR | Simplest; no scripts. Trades away conditional hard-blocking |
| Canonical instruction | `CLAUDE.md`, with `AGENTS.md` → symlink | One source, both harnesses read it |
| Canonical skills | `skills/<name>/SKILL.md`, symlinked to `.claude/skills` and `.codex/skills` | `SKILL.md` is a convention both harnesses share |
| Discipline | `shared/rules/*.md` | Harness-agnostic markdown |
| State | `.workflow/state.md` (from `state.template.md`) | Source of truth for the ship-gate checklist |

## Verified facts (this machine)

- Codex reads discipline from `AGENTS.md` (root) — symlink to `CLAUDE.md` gives one source.
- Codex repo skills are discovered under `<repo>/.codex/skills` (global: `~/.codex/skills`).
- `approval_policy` is the native, script-free way to force approval on shell actions.
- Claude Code `ask`-tier permissions natively prompt on `git push` / `gh pr create`.

## Deliberate trade (know this)

No hook conditionally blocks a commit when gates are unmet. Enforcement = advisory
(instructions) + native approval prompt on outward actions (human backstop). This starts
as simple as possible.

## Explicitly deferred (future phase, if hard blocking is wanted)

Hook-based conditional blocking (deny commit/push/PR when a `.workflow/state.md` box is
unchecked). Feasible on both harnesses, but it reintroduces adapter/trust/merge
complexity that is deliberately out of scope for this build.

## Open items to verify with a live run

- Confirm each harness discovers a **symlinked** skills dir and a symlinked root `AGENTS.md`.
- Confirm `SKILL.md` frontmatter (`name`, `description`) is accepted by both harnesses.
- If a root `AGENTS.override.md` ever exists, it takes precedence over `AGENTS.md` — the
  discipline symlink must target the active file.
