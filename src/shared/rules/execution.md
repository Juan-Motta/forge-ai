# Execution mode

How the driver executes a multi-task plan (the implementation phase of `new-feature` /
`fix-bug`). The active mode is the `Execution:` line in `PROJECT.md`, set by the codeforge
setup wizard. Default: **inline**.

## Modes

- **inline** — the driver implements each task itself, in its own turn (red → green →
  refactor per `tdd.md`). This is the default, and the only mode on Codex / OpenCode.
- **subagent-driven** _(Claude Code only)_ — the driver dispatches a **fresh subagent per
  task** to the `codeforge-implementer` agent (`.claude/agents/codeforge-implementer.md`,
  generated by the wizard at your chosen model). Each subagent implements one task TDD, runs
  the covering tests, commits, and reports back; the driver reviews between tasks. Benefits:
  isolated context per task, and cheaper/faster models can handle mechanical tasks.

## How to apply it

1. Read the `Execution:` line in `PROJECT.md`. If it is absent, use **inline**.
2. **Codex / OpenCode** always run **inline** — they have no cross-engine subagent mechanism.
3. **Claude Code + subagent-driven** — for each task in the plan, dispatch the
   `codeforge-implementer` agent (Task tool) with that task's brief; do **not** implement it
   inline. Review between tasks. Reviewer/council roles still go cross-engine per `models.md`.

Subagent-driven is a Claude-Code layer; it never changes the engine-neutral core — the same
plan runs inline everywhere else.
